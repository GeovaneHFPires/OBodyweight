Version 1.4.7

- Smoother body loading - no more stutter or "pop". Bodies now apply the way OBody does internally (the
  game updates the model on its own next pass) instead of forcing a heavy reload, so the brief freeze and
  the visible shape "snap" when an NPC's body loaded in are gone - especially noticeable in crowded areas.
- Exclude a single NPC, not just whole mods. You can now leave one specific NPC alone: add a
  "Plugin.esp|0xFormID" line to any OBodyNGWeight_Exclusions file, or bind the new "Exclude target key" in
  the MCM and press it while aiming at an NPC to toggle them on or off. Great for a custom-bodied follower
  you don't want OBW to reshape.


Version 1.4.6

- Fix: female NPCs no longer get a male body. NPCs that inherit their looks from a template (a lot of
  followers and mod-added or leveled NPCs do) were being read as male, so OBW gave them the male HIMBO body
  - and with the 1.4.5 re-assert it re-applied that wrong body every time they reloaded (cell change), which
  looked like the body "resetting" or being lost after walking between areas. OBW now resolves the real sex
  through the template, so women keep their female body and it sticks across cell changes.
- Tamer default physics (CBPC). The out-of-the-box jiggle was toned down - less exaggerated bounce on
  breasts, butt, belly and thighs, with a slightly firmer breast spring, for a more grounded default look.
  As always, you can retune it in the CBPC configs or replace it with your own.


Version 1.4.5

- Plays nicely with OBody now. If you run OBody as well, it used to overwrite OBW's bodies every time an
  NPC reloaded (walking between cells and so on) - OBW now reasserts itself on each load, so your procedural
  bodies actually stick. This only affects the Procedural and Procedural Oriented modes; OBody Sim Weight,
  and any preset you've put on an NPC by hand, are still left completely alone.
- The "exclude by plugin" list isn't capped at 128 anymore. The MCM Exclusions page is paginated now (Prev /
  Next page), so your whole load order shows up no matter how many NPC mods you run.
- New neck-seam colour fix. OBW gently matches each NPC's head tone to their body, so the neck blends in
  instead of showing a colour line. There's a "Neck color match" slider in the MCM if you want to ease it
  off or turn it off entirely.
- Tidier during normal play. The little debug pop-ups ("reprocessing...", "new seed", "regenerating body")
  no longer show up unless you've switched Debug logging on yourself.


Version 1.4.4

- MANY more female body types. Added 9 archetypes (Inverted Triangle, Diamond, Spoon, MILF, Slim Thick,
  Bottom Hourglass, Top Hourglass, Lollipop, Strongwoman) for 24 total, plus a new SHOULDER/silhouette
  axis so pear (narrow shoulders) vs inverted-triangle (broad) read distinctly. Finer shape sliders are
  now generated too (calves, torso/ribs, hip projection, breast height/cleavage, waistline...).
- Fine SHAPE traits. A rear can be heart / round / shelf / dimpled; a bust round / teardrop / wide-set /
  east-west. Each NPC may roll one of each (~30%), all derived so they stay coherent (no spikes).
- MEN now have body types too. 12 male archetypes (Lean, Fit, Swimmer, Soldier, Dadbod, Heavyset, Stocky,
  Lanky, Twink, Bodybuilder, Powerlifter, Average) + finer HIMBO shapes (shoulders, back, traps, forearms,
  calves, love handles, glutes). A differentiation pass gives men real per-region variety (far fewer
  look-alikes) and a wider lean->huge range, with a tight soft-cap so big builds never break the HIMBO mesh.
- CBPC physics now applies to MEN too (pec/belly jiggle scaled by build - firm for fit, soft for dadbod).
- Renamed body mode "OBody Presets" -> "OBody Sim Weight" (it always re-fits the preset at a per-NPC weight;
  it is NOT plain OBody). For plain OBody, just assign a preset MANUALLY through OBody's menu - OBW leaves
  manual choices untouched. Removed the confusing "NPC Default" weight mode (Weight mode = Random / Seeded).
- "Reprocess all loaded" is now mode-aware: RANDOM re-rolls every body (fresh looks each click); SEEDED
  keeps each body (just picks up changed settings/physics).


Version 1.4.3

- Renamed the "OBody Presets" body mode to "OBody Sim Weight" - a clearer name for what it actually
  does: it keeps OBody's preset SHAPE and only re-applies it at each NPC's simulated body weight, so
  NPCs sharing a preset vary in size and fullness instead of looking identical. Same behaviour, new name.
- Manual OBody assignments are now respected. When you apply a preset yourself through OBody's menu,
  OBW no longer re-fits it - only OBody's AUTO-distributed presets get the simulated weight. This keeps
  OBody's own on-demand features (anything that applies a preset to an actor on command) working as
  intended. (As before, set Weight mode to "NPC Default" for plain OBody with no simulated weight.)
- Clearer descriptions of the three body modes in the MCM and on the mod page.


Version 1.4.2

- Procedural now works with an EMPTY preset library. Previously the automatic procedural pass only ran
  after OBody distributed a preset (its "actor generated" event), so a setup with zero presets generated
  no bodies until you used the re-roll hotkey. OBW now distributes procedurally on its own: when an NPC
  loads in a Procedural mode, it gives OBody a brief moment to act and, if OBody never does (no presets,
  or an NPC OBody's own distribution skips), applies the procedural body itself. No effect in OBody
  Presets mode (that mode still relies on OBody's distribution), and excluded plugins / disabled sexes
  are still respected.


Version 1.4.1

- Exclude NPCs by mod (MCM): a new "Exclusions" page lists every plugin that adds NPCs, each with a
  checkbox - tick a mod and OBW leaves its NPCs untouched (handy for follower/NPC mods that ship
  their own hand-made bodies). Your choices are saved globally, across all saves. Applies to NPCs
  generated afterward; bodies already applied aren't reverted.
- Advanced: you can also list plugin names (one per line, .esp/.esl/.esm) in any
  SKSE\Plugins\OBodyNGWeight_Exclusions*.txt file - merged on game start, so patches can ship their
  own. Case-insensitive.
- "Generate new seed" now only affects NPCs you haven't met yet: already-seen NPCs keep their exact
  body and archetype, even after reloading. To change one specific NPC, use the re-roll hotkey.
- More body variety: NPCs of the same body type now look noticeably less alike (stronger per-NPC,
  per-region variation), while keeping smooth shapes and recognizable body types.


Version 1.4.0

New body modes
- "Procedural Oriented": generated bodies are blended toward each NPC's OBody preset, with an
  adjustable strength ("Preset orientation") - procedural variety plus the character of your presets.
- "OBody Presets" is now weight-driven: instead of every NPC that shares a preset looking identical,
  each NPC gets that preset at its own body weight, and presets with no weight range gain a
  synthesized lean<->full variation. Works for both auto-distributed and manually-assigned presets.
  Set "Weight mode" to NPC Default for plain OBody behavior.

Other
- Per-sex toggles: enable or disable OBW for female and male NPCs independently.
- More variety in Procedural mode: less repetition, including within the same body type.
- "Bias" is now a global heavier/leaner dial that affects every mode.
- MCM reorganized; options grey out when they don't apply to the selected mode.
- Debug logging toggle (off by default) under a new "Debug" section.

Fixes & performance
- Faster, more robust morph application; fixes for preset names with non-ASCII characters and a
  crash/freeze that could occur while applying presets.


Version 1.3.0

- Body archetypes: each NPC now gets one of 15 coherent body types (Balanced, Slim, Rectangle,
  Pear, Top-heavy, Hourglass, Voluptuous, Apple/Soft, BBW, Athletic, Athletic-curvy, Obese,
  Stocky, Petite, Amazon) that drives bust/waist/hips/belly/tone together. Replaces the old
  independent trait rolls as the primary shape driver (light traits still add micro-variation),
  so bodies read as intentional, recognizable shapes instead of random combinations.
- Average-centered distribution: the MEDIAN body is now balanced/average in every measure, with
  petite and curvy bodies tapering off to the sides. The old distribution pushed most NPCs to the
  extremes (the "everyone is a prominent hourglass" look) — now extremes are the minority.
- Obesity & more body variety: BBW (big soft curvy) and Obese archetypes added, plus Rectangle
  (straight/boyish), Athletic-curvy and Stocky.
- Per-body physics (CBPC, optional): physics now scales with the actual body. Bounce amplitude and
  collider size follow the body's size and softness (bigger/softer = more jiggle + larger colliders;
  toned/lean = firmer/tighter). Uses CBPC's interpolation API per NPC; soft dependency — no CBPC, no
  change. Drop-in config files included (CBPCBounceinterpolationconfig_OBW / CBPCCollisioninterpolationconfig_OBW).
- Body-type API for other mods: OBW_Native.GetArchetypeId(actor) (0-14) and GetArchetypeName(actor)
  ("Pear", "BBW", "Hourglass"...) let other plugins read an NPC's generated body type.
- MCM "Reprocess all loaded NPCs" button: re-applies the current generation + physics to every NPC
  around you without waiting for cell reloads — handy after changing settings or tuning.
- Smoother vertices retained: the soft-cap that keeps big bodies from spiking the sculpted mesh
  still applies on top of the archetype deltas (no breaks/spikes).

Version 1.2.1

Version 1.2.0

- Seam-free: OBW no longer changes an NPC's real weight value (that caused neck seams and
  body-vs-outfit weight mismatches in some setups). All body-size variety now comes from
  morphs, so head, body and outfit always match. The face keeps the NPC's editor weight.
- VR support: the re-roll hotkey now works in VR — it targets the NPC you're gazing at (an
  HMD cone-cast), falling back to yourself when nothing is in view. Desktop is unchanged
  (normal crosshair); the cone-cast only runs in VR.
- Male body tuning: tamed muscle/mass so men aren't over-built, plus a proportionality
  binding so every part scales together (no big arms on a thin chest).
- Male build slider (MCM): scale overall male corpulence (0-2x), proportions preserved.
- Performance: fewer body rebuilds per NPC — lighter on RaceMenu/SKEE overlay reprocessing.

Version 1.0.0 — Initial release

Procedural NPC weight and body randomization for OBody NG, for women (CBBE 3BA) and
men (HIMBO). Highlights:

- Per-NPC procedural bodies via SKEE morphs — no preset library needed. Deterministic
  within a playthrough (same NPC always looks the same).
- Realistic majority + tunable fantasy minority.
- Body traits (independent per-NPC features) for both sexes.
- Muscle tone: athletic women (visible definition) with a rare "snu snu" Amazon tier;
  men get tone from their build. Suppressed by body fat.
- Unusual bodies: rare out-of-distribution rolls (ultra-petite/thick, ultra-skinny/huge).
- Re-roll hotkey (re-bindable, default [ / {), clothing refit, distance-aware lazy loading.
- Full MCM: distribution mode, bias, seed, morph intensity, fantasy %, unusual %,
  unusual breasts %, athletic women %, body shape mode.
- FOMOD installer with Realistic / Balanced / Fantasy starting presets + an INI for
  new-game defaults.
- All logic in the C++ SKSE plugin; Papyrus is a thin relay.
